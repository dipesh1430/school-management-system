**Scale calculation — is 40 teachers enough for 38 sections?**

```
38 sections × 8 teachable periods/day × 6 days = 1,824 total period-slots per week

Each section actually has ~30 teacher-facing periods/week
(8 periods × 6 days = 48 raw slots minus assembly, short break, lunch = ~30–32 teaching periods)

38 sections × 30 periods = 1,140 teacher-period assignments needed per week

40 teachers × 30 periods/week capacity = 1,200 available slots

Headroom: ~60 surplus periods (5%) — very tight but workable.
```

This means **there is almost zero margin for bad teacher allocation**. If you ignore workload constraints, some teachers will hit 40+ periods/week (illegal under RTE) and others will be idle. The algorithm must be workload-aware from step one.

## 2. CBSE & RTE hard constraints (the algorithm's non-negotiable rules)

These are divided into **hard constraints** (violating them creates an invalid timetable — block generation) and **soft constraints** (violating them creates a suboptimal but technically legal timetable — warn, don't block).

### 2.1 Hard constraints (algorithm must NEVER violate these)

| Rule | Source | What to enforce in code |
|---|---|---|
| Teacher max 30–35 periods/week | RTE Act + CBSE | Reject any assignment once `teacherWorkload[id] >= 35` |
| Teacher cannot teach two sections at the same day+period | Common sense + CBSE | Index `teacherBusy[id][day][period]` — if occupied, skip |
| A section can only have one subject per period | CBSE | Index `sectionSchedule[sectionId][day][period]` |
| Lab/practical periods must be double (80–90 min continuous) | CBSE (Science/CS classes 9–12) | Never place a lab subject in Period 7 or 8 alone; always book two consecutive slots |
| Subjects must be taught only by teachers qualified for them | CBSE | Filter teachers by `teacherProfile.subjectIds.includes(subject._id)` |
| For double-shift schools: teacher's `assignedShift` must match the section's `shift` | CBSE Clause 15.3.3 | Hard filter — a Morning teacher cannot appear in an Afternoon section |
| Minimum 30-min buffer between shifts (double-shift only) | CBSE Clause 15.3.5 | No period slot exists in this window — enforce in the bell-time config, not the algorithm |

### 2.2 Soft constraints (algorithm should try hard, but warn don't block)

| Rule | Why | Code behaviour |
|---|---|---|
| No teacher teaches more than 4 periods in one day | Prevents fatigue | Prefer teachers with < 4 today; log a warning if forced to 5 |
| Same subject not scheduled two periods in a row for a section | Pedagogical | Penalize in the scoring function; allow if no alternative |
| Lab periods preferably scheduled after lunch (Periods 5–7) | Practical safety/setup | Score post-lunch lab slots higher when distributing |
| PT/Physical Education on Saturday | Standard practice | Pre-assign PT to Saturday when building the day-distribution plan |
| Core subjects (Math, English) spread across the week, not bunched | Pedagogical | Track daily distribution per subject; prefer days where this subject hasn't appeared yet |
| Homeroom/class teacher teaches Period 1 to their own section | Common school practice | Bonus score for class teacher in Period 1 of any day for their section |

### Phase 1: Validate (pre-flight — fail immediately if not met)

Run this before enqueuing the heavy work. Return errors to the UI right away.

```javascript
async function validatePreConditions(schoolId, data) {
  const errors = [];
  const warnings = [];

  // Check 1: All sections have a stream + class assigned
  const orphanSections = data.sections.filter(s => !s.classId);
  if (orphanSections.length > 0)
    errors.push(`${orphanSections.length} section(s) have no class assigned.`);

  // Check 2: All subjects have a periodsPerWeek value set
  const subjectsWithoutPeriods = data.subjects.filter(s => !s.periodsPerWeek || s.periodsPerWeek < 1);
  if (subjectsWithoutPeriods.length > 0)
    errors.push(`${subjectsWithoutPeriods.length} subject(s) missing periodsPerWeek. Set this in Subject Settings first.`);

  // Check 3: Every subject has at least one teacher qualified to teach it
  const subjectIds = new Set(data.subjects.map(s => s._id.toString()));
  const coveredSubjectIds = new Set(
    data.teachers.flatMap(t => t.subjectIds.map(id => id.toString()))
  );
  const uncoveredSubjects = [...subjectIds].filter(id => !coveredSubjectIds.has(id));
  if (uncoveredSubjects.length > 0)
    errors.push(`${uncoveredSubjects.length} subject(s) have no teacher assigned to teach them. Assign teachers in Staff Management.`);

  // Check 4: Total teacher capacity vs total periods needed
  const totalTeacherCapacity = data.teachers.length * 35; // Max 35 periods each
  const totalPeriodsNeeded = data.sections.length * 30;   // ~30 teaching periods per section per week
  if (totalPeriodsNeeded > totalTeacherCapacity)
    errors.push(`Not enough teacher capacity. Need ~${totalPeriodsNeeded} period-slots/week but have ${totalTeacherCapacity}. Add more teachers or reduce sections.`);

  // Check 5 (warning, not error): Practical subjects in Class 9–12 must have a lab room assigned
  // ... (check section.roomNumber contains "Lab" for science sections in 9–12)

  return { errors, warnings, canProceed: errors.length === 0 };
}
```

### Phase 2: Plan — build the per-section subject distribution

For each section, determine **which subjects appear on which days, in which periods, and whether they need a double slot**. This produces a flat list of "slots to fill" sorted from hardest to easiest.

```javascript
function buildSectionSlotPlan(section, subjects, subjectGroups) {
  // Get the subjects this section needs
  let requiredSubjects;
  if (section.classId.level >= 11) {
    // Use SubjectGroup for 11–12
    const group = subjectGroups.find(
      g => g.classId.equals(section.classId._id) && g.stream === section.stream
    );
    requiredSubjects = [
      ...group.compulsorySubjectIds,
      ...group.electiveSubjectIds // handled per student — for timetable, schedule all elective slots
    ];
  } else {
    // Directly from subjects collection for classes 1–10, Nursery, KG
    requiredSubjects = subjects.filter(s => s.classId.equals(section.classId._id));
  }

  // Build the slot list for the week
  const slots = [];
  for (const subject of requiredSubjects) {
    const periodsNeeded = subject.periodsPerWeek; // e.g. Math = 6, English = 5, Physics = 4
    const isDouble = subject.isPractical; // Lab = true for Science/CS

    if (isDouble) {
      // Schedule as ceil(periodsNeeded/2) double-period blocks
      const blocks = Math.ceil(periodsNeeded / 2);
      for (let i = 0; i < blocks; i++) {
        slots.push({ subjectId: subject._id, isDouble: true, periodsConsumed: 2 });
      }
    } else {
      for (let i = 0; i < periodsNeeded; i++) {
        slots.push({ subjectId: subject._id, isDouble: false, periodsConsumed: 1 });
      }
    }
  }

  // Distribute slots across 6 days (Mon–Sat)
  // PT/Sports → Saturday Period 7 or 8 (pre-assign)
  // Remaining → spread evenly, 4–5 periods/day maximum

  return slots; // Sorted: practical/double slots first, then regular
}
```

### Phase 3: Assign — the core greedy CSP loop

This is the heart of the algorithm. It processes sections in **difficulty order** (Class 12 Science first — most constraints, fewest teacher options — down to Nursery last).

```javascript
async function assignTimetable(data) {
  // --- Constraint tracking data structures ---
  // teacherBusy[teacherId][day] = Set of period numbers already booked
  const teacherBusy = {};
  // teacherDayCount[teacherId][day] = number of periods assigned today
  const teacherDayCount = {};
  // teacherWeekCount[teacherId] = total periods assigned this week
  const teacherWeekCount = {};
  // sectionDaySchedule[sectionId][day] = array of {period, subjectId, teacherId}
  const sectionDaySchedule = {};

  data.teachers.forEach(t => {
    const id = t._id.toString();
    teacherBusy[id] = { Mon:new Set(), Tue:new Set(), Wed:new Set(), Thu:new Set(), Fri:new Set(), Sat:new Set() };
    teacherDayCount[id] = { Mon:0, Tue:0, Wed:0, Thu:0, Fri:0, Sat:0 };
    teacherWeekCount[id] = 0;
  });
  data.sections.forEach(s => {
    sectionDaySchedule[s._id.toString()] = {};
    DAYS.forEach(day => { sectionDaySchedule[s._id.toString()][day] = []; });
  });

  const timetableSlots = [];  // Final output
  const conflicts = [];        // Slots that couldn't be filled

  // Sort sections hardest first
  const sortedSections = sortSectionsByDifficulty(data.sections, data.subjectGroups);

  for (const section of sortedSections) {
    const sectionId = section._id.toString();
    const sectionShift = section.shift || 'Morning'; // Default to Morning if single-shift

    // Get the slot plan for this section
    const slotPlan = buildSectionSlotPlan(section, data.subjects, data.subjectGroups);

    // Distribute slots to days (returns Map<day, slot[]>)
    const weekPlan = distributeAcrossWeek(slotPlan);

    for (const [day, daySlots] of weekPlan.entries()) {
      let periodCursor = 1; // Start at Period 1

      for (const slot of daySlots) {

        // Skip breaks — advance periodCursor past assembly/breaks
        periodCursor = skipBreaks(periodCursor, section.classId.level);

        // For double periods, ensure periods periodCursor and periodCursor+1 are both free
        if (slot.isDouble && periodCursor >= 7) {
          // Not enough room — push to next available double slot or log conflict
          conflicts.push({ sectionId, day, subject: slot.subjectId, reason: 'No room for double period' });
          continue;
        }

        // Find eligible teachers
        const eligible = data.teachers.filter(teacher => {
          const tId = teacher._id.toString();
          return (
            teacher.subjectIds.some(s => s.equals(slot.subjectId)) &&   // Teaches this subject
            !teacherBusy[tId][day].has(periodCursor) &&                  // Free this period
            teacherWeekCount[tId] < 35 &&                                // Under weekly limit
            teacherDayCount[tId][day] < 5 &&                             // Under daily limit (soft: prefer <4)
            (!section.shift || teacher.assignedShift === sectionShift)   // Shift match (double-shift schools)
          );
        });

        if (eligible.length === 0) {
          conflicts.push({
            sectionId,
            sectionName: `${section.classId.name} ${section.stream || ''} Sec ${section.sectionName}`,
            day,
            period: periodCursor,
            subjectId: slot.subjectId,
            reason: 'No eligible teacher available'
          });
          periodCursor += slot.periodsConsumed;
          continue;
        }

        // Score eligible teachers and pick the best
        const best = scoreAndPick(eligible, day, teacherWeekCount, teacherDayCount, section);

        // Register the assignment
        const bId = best._id.toString();
        teacherBusy[bId][day].add(periodCursor);
        if (slot.isDouble) teacherBusy[bId][day].add(periodCursor + 1);
        teacherWeekCount[bId] += slot.periodsConsumed;
        teacherDayCount[bId][day] += slot.periodsConsumed;

        timetableSlots.push({
          schoolId: section.schoolId,
          sectionId: section._id,
          subjectId: slot.subjectId,
          teacherId: best._id,
          day,
          periodNumber: periodCursor,
          isDouble: slot.isDouble,
          endPeriod: slot.isDouble ? periodCursor + 1 : periodCursor
        });

        periodCursor += slot.periodsConsumed;
      }
    }
  }

  return { timetableSlots, conflicts };
}
```

### Phase 4: Repair — fix soft constraint violations

After the greedy pass, scan the draft for soft-constraint violations and attempt local swaps to fix them. This is lightweight — only run if conflicts count is above 0.

```javascript
async function repairDraft(timetableSlots, conflicts, teacherBusy, teacherWeekCount) {
  // Repair strategy: for each conflict, try swapping two existing assignments
  // that would satisfy both the conflict and not create a new hard constraint violation.

  for (const conflict of conflicts) {
    // Find any slot already assigned where:
    // 1. The teacher is the same subject
    // 2. The teacher's swap destination has no clash
    // 3. The section's original period becomes free

    const candidate = timetableSlots.find(slot =>
      slot.subjectId.equals(conflict.subjectId) &&
      slot.sectionId.toString() !== conflict.sectionId &&
      !teacherBusy[slot.teacherId.toString()][conflict.day].has(conflict.period)
    );

    if (candidate) {
      // Perform swap: assign this teacher to the conflicted slot
      // Free up the original slot for the section that candidate came from (to be re-attempted)
      timetableSlots.push({
        sectionId: conflict.sectionId,
        subjectId: conflict.subjectId,
        teacherId: candidate.teacherId,
        day: conflict.day,
        periodNumber: conflict.period,
        isDouble: conflict.isDouble
      });
      // Remove the conflict
      conflicts.splice(conflicts.indexOf(conflict), 1);
    }
  }

  return { timetableSlots, unresolvedConflicts: conflicts };
}
```

## 6. Scoring function — how to pick the best teacher when multiple are eligible

When multiple teachers are eligible for a slot, use this scoring function rather than picking randomly. Higher score = better pick.

```javascript
function scoreAndPick(eligibleTeachers, day, teacherWeekCount, teacherDayCount, section) {
  const scored = eligibleTeachers.map(teacher => {
    const tId = teacher._id.toString();
    let score = 100; // Start with base score

    // BONUS: Teacher is below average workload — prefer underutilized teachers
    const avgWeekLoad = 30;
    if (teacherWeekCount[tId] < avgWeekLoad) score += 15;

    // BONUS: Teacher has fewer periods today (prefer balanced daily load)
    if (teacherDayCount[tId][day] <= 2) score += 20;
    else if (teacherDayCount[tId][day] === 3) score += 10;
    else if (teacherDayCount[tId][day] >= 4) score -= 30; // Penalty: getting close to daily limit

    // BONUS: Teacher is the class teacher for this section
    if (section.classTeacherId && teacher._id.equals(section.classTeacherId)) score += 25;

    // BONUS: Teacher's stream specialization matches the section's stream
    if (section.stream && teacher.streamSpecialization === section.stream) score += 15;

    // PENALTY: Weekly workload approaching limit
    if (teacherWeekCount[tId] >= 30) score -= 20;
    if (teacherWeekCount[tId] >= 33) score -= 40; // Danger zone

    return { teacher, score };
  });

  // Pick highest score; randomly break ties to avoid always picking the same teacher
  scored.sort((a, b) => b.score - a.score);
  const topScore = scored[0].score;
  const topCandidates = scored.filter(s => s.score === topScore);
  return topCandidates[Math.floor(Math.random() * topCandidates.length)].teacher;
}
```

## 7. Difficulty ordering: why hardest sections go first

The greedy algorithm gets worse options as teacher capacity is consumed. Always schedule the most constrained sections first so they get the widest pick of eligible teachers.

**Foundational sections (Nursery/LKG/UKG) are NOT passed to this function at all** — they go to the separate `generateFoundationalTimetable()` track in §16 and use a completely different, simpler scheduling model. This function only receives `mainSections` (Class 1–12).

```javascript
function sortSectionsByDifficulty(sections, subjectGroups) {
  // sections here is ONLY mainSections (Class 1–12) — Foundational already separated
  return sections.sort((a, b) => {
    // Factor 1: Higher class level = more constrained subjects = schedule first
    const levelDiff = (b.classId.level || 0) - (a.classId.level || 0);
    if (levelDiff !== 0) return levelDiff;

    // Factor 2: Science > Commerce > Arts > General (Science has fewest eligible teachers for Physics/Chem)
    const streamOrder = { Science: 3, Commerce: 2, Arts: 1, null: 0, undefined: 0 };
    const streamDiff = (streamOrder[b.stream] || 0) - (streamOrder[a.stream] || 0);
    if (streamDiff !== 0) return streamDiff;

    // Factor 3: If double shift, Morning shift first (more teachers typically assigned there)
    if (a.shift === 'Morning' && b.shift !== 'Morning') return -1;

    return 0;
  });
}
```

Concretely for your 32 main sections (Class 1–12), the CSP order is:

3. Class 12 Arts A/B
4. Class 11 Science A/B

## 2. CBSE & RTE Hard Constraints

These are divided into **hard constraints** (violating them creates an invalid timetable — block generation) and **soft constraints** (violating them creates a suboptimal but technically legal timetable — warn, don't block)[cite: 1].

### 2.1 Hard Constraints (Algorithm must NEVER violate these)
* **Teacher Weekly Limit:** Reject any assignment once `teacherWorkload[id] >= 35` (RTE Act + CBSE mandate)[cite: 1].
* **No Double-Booking:** Index `teacherBusy[id][day][period]` — a teacher cannot be in two rooms at once[cite: 1].
* **Single Subject per Slot:** Index `sectionSchedule[sectionId][day][period]`[cite: 1].
* **Double Periods for Labs:** Science and Computer practicals (Classes 9–12) must be scheduled as 80–90 minute continuous blocks (never place a lab in Period 8 alone)[cite: 1].
* **Strict Qualification Mapping:** Filter teachers by `teacherProfile.subjectIds.includes(subject._id)`[cite: 1]. PRTs teach 1–5, TGTs teach 6–10, PGTs teach 11–12, and NTTs teach Foundational[cite: 1].
* **Shift Isolation (Double-Shift Schools):** A Morning shift teacher (`assignedShift === 'Morning'`) cannot be scheduled into an Afternoon section (CBSE Clause 15.3.3)[cite: 1].
* **Sanitization Buffer:** Minimum 30-minute operational gap between Morning dispersal and Afternoon assembly (CBSE Clause 15.3.5)[cite: 1].

### 2.2 Soft Constraints (Optimize during scoring and repair)
* **Daily Fatigue Cap:** Prefer teachers with < 4 periods assigned today; log a warning if forced to 5[cite: 1].
* **Subject Spreading:** Prevent scheduling the same academic subject two periods in a row for a section unless it is a designated practical lab[cite: 1].
* **Post-Lunch Labs:** Score Periods 5–7 higher when distributing science practicals for safety and setup convenience[cite: 1].
* **Weekend Physical Education:** Pre-assign PT / Sports slots to Saturday morning periods[cite: 1].
* **Class Teacher Anchor:** Add a bonus score if a homeroom teacher is assigned to Period 1 for their own section[cite: 1].

async function assignMainTimetable(mainSections, mainTeachers, subjects, subjectGroups) {
  const teacherBusy = {};
  const teacherWeekCount = {};
  
  mainTeachers.forEach(t => {
    teacherBusy[t._id.toString()] = { Mon: new Set(), Tue: new Set(), Wed: new Set(), Thu: new Set(), Fri: new Set(), Sat: new Set() };
    teacherWeekCount[t._id.toString()] = 0;
  });

  const timetableSlots = [];
  const conflicts = [];
  const sortedSections = sortSectionsByDifficulty(mainSections);

  for (const section of sortedSections) {
    const slotPlan = buildSectionSlotPlan(section, subjects, subjectGroups);
    const weekPlan = distributeAcrossWeek(slotPlan);

    for (const [day, daySlots] of weekPlan.entries()) {
      let periodCursor = 1;

      for (const slot of daySlots) {
        periodCursor = skipBreaks(periodCursor, section.classId.level);

        const eligible = mainTeachers.filter(teacher => {
          const tId = teacher._id.toString();
          return (
            teacher.subjectIds.some(s => s.equals(slot.subjectId)) &&
            !teacherBusy[tId][day].has(periodCursor) &&
            teacherWeekCount[tId] < 35 &&
            (!section.shift || teacher.assignedShift === section.shift)
          );
        });

        if (eligible.length === 0) {
          conflicts.push({ sectionId: section._id, day, period: periodCursor, subjectId: slot.subjectId, reason: 'No eligible teacher' });
          periodCursor += slot.periodsConsumed;
          continue;
        }

        const bestTeacher = scoreAndPick(eligible, day, teacherWeekCount, section);
        const bId = bestTeacher._id.toString();

        teacherBusy[bId][day].add(periodCursor);
        if (slot.isDouble) teacherBusy[bId][day].add(periodCursor + 1);
        teacherWeekCount[bId] += slot.periodsConsumed;

        timetableSlots.push({
          schoolId: section.schoolId,
          sectionId: section._id,
          subjectId: slot.subjectId,
          teacherId: bestTeacher._id,
          day,
          periodNumber: periodCursor,
          isDouble: slot.isDouble
        });

        periodCursor += slot.periodsConsumed;
      }
    }
  }
  return { timetableSlots, conflicts };
}
6. Scoring Function (Teacher Selection)
When multiple teachers are eligible for a period, the worker evaluates candidates using a weighted scoring matrix[cite: 1]:

JavaScript
function scoreAndPick(eligibleTeachers, day, teacherWeekCount, section) {
  const scored = eligibleTeachers.map(teacher => {
    const tId = teacher._id.toString();
    let score = 100;

    // Favor underutilized teachers to balance workload across the department
    if (teacherWeekCount[tId] < 25) score += 20;
    
    // Prioritize homeroom teacher for their own class
    if (section.classTeacherId && teacher._id.equals(section.classTeacherId)) score += 25;
    
    // Prioritize stream specialization match (e.g., Commerce PGT for 11th Commerce)
    if (section.stream && teacher.streamSpecialization === section.stream) score += 15;
    
    // Penalize teachers approaching their legal RTE weekly workload limit
    if (teacherWeekCount[tId] >= 30) score -= 25;
    if (teacherWeekCount[tId] >= 33) score -= 50;

    return { teacher, score };
  });

  scored.sort((a, b) => b.score - a.score);
  return scored[0].teacher;
}
7. Difficulty Ordering (Most Constrained First)
To prevent resource starvation, senior secondary streams must be scheduled before middle and primary classes[cite: 1]. Foundational classes are excluded from this function entirely[cite: 1].

JavaScript
function sortSectionsByDifficulty(mainSections) {
  return mainSections.sort((a, b) => {
    // 1. Sort by Grade Level Descending (12th down to 1st)
    const levelDiff = (b.classId.level || 0) - (a.classId.level || 0);
    if (levelDiff !== 0) return levelDiff;

    // 2. Sort by Stream Complexity (Science > Commerce > Arts > General)
    const streamOrder = { Science: 3, Commerce: 2, Arts: 1, General: 0 };
    const streamDiff = (streamOrder[b.stream] || 0) - (streamOrder[a.stream] || 0);
    if (streamDiff !== 0) return streamDiff;

    // 3. Sort Morning Shift before Afternoon Shift
    if (a.shift === 'Morning' && b.shift !== 'Morning') return -1;
    return 0;
  });
}

Queries all class periods mapped to the absent teacher for the current day.

Sends a push notification to the selected substitute's mobile app upon administrative confirmation.

Foundational Timetable Generator Logic
This function runs in parallel with the main CSP engine[cite: 1]. It assigns dedicated NTT teachers to their respective pre-primary sections without querying the subject or subject-group collections[cite: 1]:

JavaScript
async function generateFoundationalTimetable(foundationalSections, nttTeachers) {
  const timetableSlots = [];
  const conflicts = [];
  const DAYS_FOUNDATIONAL = ['Mon', 'Tue', 'Wed', 'Thu', 'Fri']; // No Saturday classes

  for (const section of foundationalSections) {
    // Find the dedicated class teacher or pull an available NTT staff member
    let assignedTeacher = nttTeachers.find(t => t._id.equals(section.classTeacherId));
    
    if (!assignedTeacher && nttTeachers.length > 0) {
      // Fallback: assign the first available NTT teacher not already over-allocated
      assignedTeacher = nttTeachers[0];
    }

    if (!assignedTeacher) {
      conflicts.push({
        sectionId: section._id,
        reason: 'No NTT (Nursery Teacher Training) qualified teacher available for Foundational section.'
      });
      continue;
    }

    // Populate the 5-day schedule from the fixed activity template
    for (const day of DAYS_FOUNDATIONAL) {
      for (const block of FOUNDATIONAL_TEMPLATE) {
        // Skip non-teaching break slots from database assignment records
        if (['Social_Break', 'Meal_Break'].includes(block.category)) continue;

        timetableSlots.push({
          schoolId: section.schoolId,
          sectionId: section._id,
          subjectName: block.activity, // Stores activity name directly in place of subjectId
          teacherId: assignedTeacher._id,
          day,
          periodNumber: block.periodNumber,
          isDouble: false,
          timeSlot: block.time,
          stageType: 'Foundational'
        });
      }
    }
  }

  return { timetableSlots, conflicts };
}
Key Advantages of this Track Isolation:

Zero Conflict with Senior Streams: NTT teachers are contained within the generateFoundationalTimetable function[cite: 1]. They are never evaluated during the Class 11–12 Science or Commerce assignment loops[cite: 1].

No Subject Setup Overhead: School administrators do not need to manually configure periodsPerWeek or practical lab flags for 3-year-olds in the settings panel[cite: 1].

Automatic CBSE Compliance: Fulfills the NEP 2020 mandate for play-based foundational learning while keeping the database schema unified under the same timetabledrafts collection[cite: 1].
